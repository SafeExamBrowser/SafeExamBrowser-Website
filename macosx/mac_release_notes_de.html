Release Notes für Safe Exam Browser version 3.6.1:

SEB kann nun Bedienungshilfen wie VoiceOver steuern. Mit dem optionalen SEB Server und seinem Moodle-Plugin sind mehrere aufeinanderfolgende Quizzes möglich.

Neu in SEB 3.6 (Build 15726):

  • Erlaubt das Öffnen der Einstellungen beim ersten Start von SEB/mit den Standard-Client-Einstellungen. Dies löst das Problem des Zugriffs auf die Einstellungen, wenn Sie SEB manuell einrichten möchten (da die Einstellung „Einstellungen-Fenster darf auf Client geöffnet werden” seit SEB 3.6 standardmässig deaktiviert/falsch ist).
  • Die Funktion „Automatisches Ausfüllen“ wird nun in bestimmten Situationen blockiert.
  • Google Chrome wurde zu den voreingestellten standardmässig verbotenen Prozessen hinzugefügt. Chrome wird nun von SEB beendet, wenn der AAC-Bewertungsmodus nicht verwendet wird. Dadurch wird verhindert, dass SEB den Prozess „Chromium Helper“ ständig beenden muss.
  • Behoben: Die Einstellungen „Öffnen/Speichern-Fenster zulassen” und „Share Sheet zulassen” (bei der Arbeit mit zugelassenen zusätzlichen Anwendungen) funktionierten nicht. 
  • Beendet den Öffnen/Speichern-Fenster-Agenten nicht mehr, während die Einstellungen geöffnet sind („Öffnen” und „Speichern unter” funktionieren in den Einstellungen auch dann, wenn der AAC-Bewertungsmodus aktiv ist und die Einstellungen „Öffnen/Speichern-Fenster zulassen” und „Share Sheet zulassen” nicht aktiviert sind). 
  • Jetzt wird nur noch um die Erlaubnis zum Herunterladen/Zugriff auf den Protokollordner gebeten, wenn das Herunterladen/Protokollieren in den aktuellen Einstellungen aktiviert ist.
  • Absturz beim Laden von Webseiten mit den Einstellungen sendBrowserExamKey = true und logLevel = Verbose behoben.

Optionale Funktionen, die individuell konfiguriert werden müssen und standardmässig deaktiviert sind

  • Behoben: Einstellungen für die Bedienungshilfen (wie das Aktivieren/Deaktivieren von VoiceOver) wurden bei der Neukonfiguration mit dem SEB-Server nicht aktualisiert.
  • Fehlerbehebung beim Starten einer SEB-Serverprüfung über die Verbindungskonfiguration (Prüfungsliste), bei der SEB 3.6 keinen App-Signaturschlüssel (ASK) gesendet hat.
Release Notes für Safe Exam Browser version 3.6:

SEB kann nun Bedienungshilfen wie VoiceOver steuern. Mit dem optionalen SEB Server und seinem Moodle-Plugin sind mehrere aufeinanderfolgende Quizzes möglich.

WICHTIG: Wenn Sie Konfigurationsdateien mit SEB 3.5 (macOS oder iOS) erstellt oder geöffnet und gespeichert haben, sollten Sie den AppleSpell-Prozess aus diesen Einstellungen (SEB-Einstellungen / Anwendungen / Verbotene Prozesse) in SEB 3.6 entfernen und die Konfigurationsdatei erneut speichern. Andernfalls kann es bei einigen macOS-Versionen zu Verzögerungen bei der Texteingabe kommen (siehe Release Notes SEB 3.5.1).

Neu in SEB 3.6 (Build 156D0):

  • Optionen in den In-App-Einstellungen / Benutzeroberfläche hinzugefügt, um die Bedienungshilfe VoiceOver zu steuern (Systemstandard, aktivieren, deaktivieren).
  • Die Standardeinstellung für „Öffnen des Einstellungsfensters auf dem Client zulassen“ wurde auf deaktiviert geändert.
  • Die Einstellung für die minimal unterstützte macOS-Version (SEBMinMacOSVersionSupported) wurde auf macOS 11 Big Sur geändert. Wenn Sie diese Einstellung manuell ändern, sollte SEB 3.6 weiterhin unter macOS 10.13 bis 10.15 laufen, aber es ist sehr wahrscheinlich, dass die mit diesen macOS-Versionen ausgelieferte WebKit-Browser-Engine nicht vollständig mit einigen Webanwendungen kompatibel ist und unter bestimmten Umständen sogar abstürzen kann.
  • Hardkodierten User Agent String aktualisiert (3.5.4 zusätzlich zu 3.6).

  • Jetzt werden Prüfungen mit mehreren aufeinanderfolgenden Quizzes unterstützt, wenn der optionale SEB Server und das SEB-Server-Moodle-Plugin verwendet werden.
  • Versuch, einen Absturz beim Herstellen einer SEB-Server-Verbindung zu beheben. Wenn Sie SEB Server verwenden und beim Herstellen einer Verbindung dazu Abstürze auftreten, SENDEN SIE UNS BITTE EINEN FEHLERBERICHT (einschliesslich der entsprechenden SEB- Logdateien und Ihrer Konfigurationsdatei). Siehe Support-Link.
  • Behebung von zufälligen Abstürzen bei der Bildschirmüberwachung.

Neu in SEB 3.5.4 (Build 1567A):

  • Behoben: Der App-Signaturschlüssel (ASK) wurde bei Verwendung des optionalen, individuell konfigurierten SEB-Servers nicht übertragen.
  • Abstürze unter macOS 10.15: Das moderne WebView (WKWebView) stürzt leider beim Laden einiger Seiten unter macOS 10.15 ab. Daher müssen wir die Standardeinstellung für die minimal unterstützte macOS-Version auf 11 setzen. Wenn Sie macOS 10.15 unterstützen müssen, können Sie möglicherweise das klassische WebView in Ihren SEB-Einstellungen verwenden (was jedoch nicht empfohlen wird, da es veraltet ist).

Release Notes für Safe Exam Browser version 3.5.1:

SEB 3.5.1 bietet neue Optionen zum Herunterladen und Öffnen bestimmter Dateitypen mit zusätzlichen Apps, verbessert die Benutzerfreundlichkeit einiger Einstellungsoptionen und behebt Verzögerungen bei der Texteingabe sowie Sicherheitsprobleme.

WICHTIG: Wenn Sie Konfigurationsdateien mit SEB 3.5 (macOS oder iOS) erstellt oder geöffnet und gespeichert haben, sollten Sie den AppleSpell-Prozess aus diesen Einstellungen (SEB-Einstellungen / Anwendungen / Verbotene Prozesse) in SEB 3.5.1 entfernen und die Konfigurationsdatei erneut speichern. Andernfalls kann es bei einigen macOS-Versionen zu Verzögerungen bei der Texteingabe kommen (siehe unten).

Neu in SEB 3.5.1 (Build 15670):

  • Konfigurierte Dateitypen herunterladen (und optional in einer bestimmten App öffnen), anstatt sie im SEB-Browser anzuzeigen: Unter „Einstellungen / Downloads/Uploads“ wurde eine neue Liste ‚Dateitypen‘ hinzugefügt, in der Sie Dateitypen festlegen können, die heruntergeladen statt im Browser angezeigt werden sollen. Sie können auch die zugehörige zusätzliche App für den Dateityp angeben und die Option „Dateien nach dem Herunterladen öffnen“ verwenden.
    Die neuen Einstellungs-Keys sind: downloadFileTypes (Array von Dictionaries) mit Dateitypen mit den Properties „extension“ (Dateiendung), ‚os‘ und „associatedAppId“ (Bundle-Identifier der App, die nach Abschluss des Downloads geöffnet wird). Derzeit in SEB für macOS und iOS unterstützt.
  • Behoben: Das Blockieren von Inline-Vorschlägen in WebView-Textfeldern führte bei bestimmten macOS-Versionen und Systemeinstellungen zu Verzögerungen bei der Texteingabe: Das Beenden des AppleSpell-Prozesses wurde entfernt und eine bessere Alternative implementiert. Hinweis: Wenn Sie SEB-Konfigurationen mit SEB 3.5 erstellt haben, sollten Sie den AppleSpell-Prozess aus diesen Einstellungen entfernen (SEB-Einstellungen / Anwendungen / Verbotene Prozesse).
  • Behoben: Einfrieren bei Verwendung des AAC-Bewertungsmodus, nachdem das Dialogfeld für die Download-Ordnerberechtigungen angezeigt wurde. Jetzt wird eine Vorabprüfung der Systemeinstellungen für Datenschutz Dateien und Ordnerzugriffsberechtigungen für Download- und Logdateiordner durchgeführt.
  • Unterstützung der Einstellung „Als Hintergrund-Prozess zulassen“ für erlaubte Prozesse: Erlaubt es dem erlaubten Prozess, bereits beim Start von SEB zu laufen (wird nicht beendet, wenn die Session in SEB gestartet und beendet wird).
  • Aktualisierte Benutzeroberfläche und Formulierungen im Abschnitt „Prüfungseinstellungen“ und verbesserte Benutzerfreundlichkeit für die Freigabe von BEK/CK (auch aktiviert, wenn die umformulierte Einstellung „Browser- und Konfigurationsschlüssel im HTTP-Header senden“ deaktiviert ist).
  • Blockiert nun den Systemprozess DataDetectorsViewService, um zu verhindern, dass in einigen Fällen Kontakte oder Kalender-Popups angezeigt werden.
  • Es werden keine leeren Inhaltsfilterregel-Listen mehr kompiliert, wenn das moderne WebView (WKWebView) verwendet wird.
  • Mögliche Ursache für Absturz behoben (trat auf, wenn die SEB-Einstellungen beim Start zurückgesetzt wurden).
  • SEB 3.5 zu den Versions-User-Agent-Strings hinzugefügt.
  • Voreingestellte verbotene Prozesse aktualisiert (Win/Mac).
  • Standardeinstellungen für allowUploads (false) / allowDownUploads (true, veraltet: separate Einstellungen für allowDownload/allowUploads verwenden) geändert.

Optionale Funktionen, die individuell konfiguriert werden müssen und standardmässig deaktiviert sind

  • Neustart anstelle des Beendens der SEB Server-Prüfungssitzung funktioniert nicht: Ein Neustart der SEB Server-Prüfungssitzungen ist nicht mehr möglich.
  • Der Übertragungsstatus „Alle zwischengespeicherten Screenshots übertragen“ wird nicht mehr im Dock-Icon angezeigt, wenn das Token erneuert wurde.
Release Notes für Safe Exam Browser version 3.5:

SEB 3.5 unterstützt den URL-Content-(Inhalts-) Filter auch in der modernen Browser-Engine, behebt allgemeine Probleme und in der optionalen Screen Proctoring Funktion (Bildschirm-Fernüberwachung, erfordert den Betrieb einer eigenen Instanz von SEB Server mit SEB Screen Proctoring Service).

Neu in SEB 3.5 (Build 15487):

  • Unterstützung für URL-Inhaltsfilter im modernen WebView (WKWebView) hinzugefügt. Hinweis: Wenn benutzerdefinierte reguläre Ausdrücke als SEB-URL-Filterausdrücke verwendet werden, ist nur die von WKWebView/Safari Content Filters unterstützte Regex-Teilmenge erlaubt, siehe https://webkit.org/blog/3476/content-blockers-first-look/
  • Aktualisierte Auswahlrichtlinie für das moderne WebView (nicht mehr abhängig von der Verwendung von URL-Inhaltsfiltern).
  • Umbenennung der Webbrowser-Engine- (WebView-) Richtlinie 'Moderne bevorzugen' in 'Moderne erzwingen' und Entfernung von UI-Verweisen auf URL-Inhaltsfilter in Bezug auf die WebView-Auswahlrichtlinie.
  • AI-Schreibwerkzeuge in macOS Sequoia werden jetzt blockiert.
  • Blockiert die Inline-Textvorhersage.
  • Entfernt jetzt das Passwörter-Menüleisten-Element.
  • Schaltfläche "Applikation/Executable auswählen" auch für verbotene macOS-Apps hinzugefügt (Einstellungen / Applikationen).
  • Behoben: Die Eingabe einer URL-Filterregel (Einstellungen / Netzwerk / Filter) im Textfeld musste bestätigt werden ("Eingabe/Return"-Taste).
  • Behoben: Neue URL-Filterregel wurde nicht gespeichert, wenn "Anwenden und Neustart" verwendet wurde.
  • Start-URL mit offensichtlich falschem Format in Einstellungen / Allgemein (z.B. mit vorangestelltem Leerzeichen) wird nicht mehr akzeptiert und verhindert, dass SEB bei einigen falschen Start-URLs abstürzt.
  • Verbesserte Validierung der Start- und SEB-Server-URL in Einstellungen / Allgemein: Wenn kein Protokollschema eingegeben wird, wird https:// hinzugefügt.
  • Behoben: ConfigKey wurde im Settings-Fenster nach dem Speichern einer Konfigurationsdatei nicht korrekt aktualisiert.
  • Der Standardwert für die minimal unterstützte macOS-Version wurde auf 10.15 geändert. SEB könnte noch mit macOS 10.13 und 10.14 funktionieren, wenn Sie diese Einstellung ändern, aber wir testen nicht mit diesen Versionen und wie Apple empfehlen wir generell, nur die drei neuesten macOS-Versionen zu verwenden.
  • Einige ältere SEB-Versionen wurden aus dem User-Agent-String entfernt (es werden nur noch 3.4, 3.4.1 und 3.5 ausgegeben).
  • Die Safari-Version in hartkodierten User-Agent-Element-Strings wurde aktualisiert.
  • Erlaubt nun die unabhängige Aktivierung/Deaktivierung von Bildschirm-/ Fensteraufnahme in Einstellungen / Sicherheit.

Optionale Funktionen, die individuell konfiguriert werden müssen und standardmässig deaktiviert sind

  • Die SEB Server-Funktion „Hand heben“ ist nun wieder verfügbar, wenn sie in den Session-Einstellungen aktiviert ist.
  • Behoben: Das Präfix in der Moodle Login User ID Auflösung für SEB Server funktionierte nicht.
  • Screen Proctoring Metadaten: Jetzt wird die SEB-Version im fokusierten und in der Liste der Fenstertitel der Anwendung entfernt.
  • Behoben: Abgelaufene Screen Proctoring Token verursachten unnötige Unterbrechungen der Serververbindung.
  • Behoben: SEB konnte nicht beendet werden, weil versucht wurde, einen Screenshot zu übertragen, der bereits übertragen und gelöscht worden war.
  • Verbesserte Sicherheit beim Zwischenspeichern von Screen Shots, wenn die Verbindung offline ist.
  • Nun wird zuerst Screen Proctoring geschlossen, dann die SEB Serververbindung.

Release Notes für Safe Exam Browser version 3.4:

SEB 3.4 erlaubt die Verwendung von Drittanbieter-Applikationen in Prüfungen, behebt Probleme, die vor allem unter macOS 14 Sonoma auftreten können, und implementiert die optionale Screen-Proctoring-Funktion.

Neu in SEB 3.4 (Build 15396):

  • Unterstützung für die Ausführung von Drittanbieterprogrammen im macOS AAC Multi App Assessment Modus (erfordert macOS 12 oder neuer).
  • Optionales temporäres Upload- und Download-Verzeichnis, um den freien Zugriff auf das gesamte Dateisystem zu verhindern, wenn zusätzliche Apps verwendet werden.
  • Ein mögliches Problem wurde behoben, bei dem SEB manchmal kein Fenster öffnete und mit cmd-Q beendet werden musste. Dies scheint häufiger/nur unter macOS 14 Sonoma zu passieren.
  • Die Abfrage von Berechtigungen für Webanwendungen zum Zugriff auf Mikrofon und Kamera wurde behoben.
  • Problem mit der Blockierung von Bildschirmfotos unter macOS 14 behoben.
  • Injizieren von ConfigKey und BrowserExamKey in alle iFrames, was jedoch nur beim Laden der Seite korrekt funktioniert, wenn die Funktion SafeExamBrowser.security.updateKeys(function callback) verwendet wird (in der Callback-Funktion können Sie die korrekten CK/BEK-Schlüssel für den Frame lesen).
  • Beendete Nicht-UI-Prozesse werden nicht mehr neu gestartet.
  • Unterstützung für Screen Proctoring (Bildschirm-Fernüberwachung). Diese bietet eine Live-Ansicht der Bildschirme von angeschlossenen SEB-Clients und Aufzeichnungen mit verschiedenen durchsuchbaren Metadaten. Diese Funktion erfordert den Betrieb einer eigenen Instanz von SEB Server mit SEB Screen Proctoring Service. SEB verbindet sich niemals mit einem zentralisierten Proctoring- oder anderen Servern, dies muss immer individuell konfiguriert werden, falls Sie es nutzen wollen.
  • Jetzt werden generische Prüfungssysteme mit SEB Server unterstützt: Sie können die URL einer beliebigen webbasierten Prüfung/eines beliebigen Tests/Quiz eingeben und SEB Server verwenden, um die in seinem Web-Backend erstellten SEB-Einstellungen bereitzustellen. Sie können auch die angeschlossenen SEB Clients überwachen, inklusive einer automatischen Überprüfung der Integrität der einzelnen SEB Clients.

Bekannte Einschränkungen

Bitte melden Sie uns Fehler und Wünsche für neue Funktionen. Sie können (trotz der englischen Sourceforge-Seite) Ihre Kommentare in Deutsch hinterlassen. Bitte immer angeben, welche SEB-Version und welches Betriebssystem (OS X 10.11, 10.10, 10.9, 10.8, 10.7, 10.6) Sie einsetzen, eventuell auch Details zu Ihrem Rechner.

  • SEB unterbindet keine Zugriffe auf den Computer über das Netzwerk. Darum sollten alle entsprechenden Freigaben für System- oder Drittanbieter-Netzwerkdienste wie Bildschirmfreigabe, Dateifreigabe, entfernte Anmeldung und entfernte Verwaltung abgeschaltet oder mit (dem Benutzer nicht bekannten) Administrator-Passwörtern geschützt werden. Oder es könnte auch eine externe Firewall mit sehr restriktiven Zugriffsregeln benutzt werden.
  • SEB hindert andere Programme nicht, Audio abzuspielen. Sie sollten deshalb unter Umständen alle VOIP/Audio-Kommunikationsdienste wie iChat und Skype abschalten, insbesondere bei Prüfungen mit Audio-Inhalten, bei denen die Prüflinge Kopfhörer benutzen dürfen.
  • Falls Ihre Prüfung das https-Protokoll oder Java-Applets verwendet, die sich mit sicheren Servern verbinden, sollten Sie sicherstellen, dass den verwendeten Zertifikaten auf den benutzten Computern vertraut wird. Falls dies nicht der Fall ist, können Sie die Zertifikate entweder manuell zum macOS Schlüsselbund hinzufügen oder im Zertifikat akzeptieren-Dialog das Häkchen bei ... immer vertrauen (wird erst nach Betätigen der Schaltfläche Zertifikat einblenden angezeigt) setzen.


Diese meisten dieser Einschränkungen sollten in zukünftigen Versionen von SEB für macOS beseitigt werden.
